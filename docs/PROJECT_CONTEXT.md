# PROJECT_CONTEXT

## 1. 프로젝트 비전

MoIja는 축구/풋살 동호회 운영을 위한 SaaS 플랫폼이다. 단순 경기 관리 앱이 아니라, 신뢰도 기반 팀 문화 형성과 데이터 기반 팀 운영을 목표로 한다.

### 핵심 목표

- 참여율 향상
- 노쇼 감소
- 신뢰도 기반 팀 문화 형성
- 운영자 업무 최소화
- 데이터 기반 팀 운영

### 서비스 방향

- 초기 타겟: 축구, 풋살
- 향후 확장: 러닝 크루, 스터디 모임, 독서 모임, 기타 정기 모임
- 설계 원칙: DB 및 시스템 구조는 특정 스포츠에 종속되지 않도록 설계

## 2. 기술 스택

| 영역 | 기술 |
| --- | --- |
| Frontend | Next.js, TypeScript, Tailwind CSS, PWA |
| Backend | Supabase, PostgreSQL |
| 배포 | GitHub + Vercel |

## 3. 사용자 역할 및 권한

| 역할 | 권한 |
| --- | --- |
| Owner | 팀 생성/설정, 경기 생성, 출석 확정, 경기 결과 입력, 권한 관리 |
| Manager | 경기 생성, 출석 관리, 경기 결과 입력 |
| Coach | 포지션 지정, 라인업 구성, 선수 관리 |
| Member | 참석 신청, 통계 조회, 개인 기록 조회 |

## 4. 로그인 정책

- 방식: Kakao 로그인
- 저장 정보: Kakao ID, 닉네임, 프로필 이미지
- 미저장 정보: 전화번호, 주민등록번호, 주소, 생년월일

## 5. 출석 시스템

### 지원 방식

| 방식 | 설명 |
| --- | --- |
| 수동 출석 | 운영자가 직접 체크 |
| QR 출석 | QR 스캔으로 처리 |
| GPS 출석 | 현재 위치 확인 |
| GPS + 운영자 승인 | GPS 확인 후 운영자 최종 승인 |

### 출석 상태

- 참석
- 불참
- 대기
- 노쇼

### 장소 관리

- 자유 텍스트 입력 금지
- Kakao Maps 또는 Naver Maps 기반 장소 선택
- 저장 데이터: 장소명, 주소, 위도, 경도
- 홈구장 즐겨찾기 지원

## 6. 신뢰도 시스템

측정 요소는 참석률, 노쇼율, 연속 출석, 지각, 취소 시점이다. MVP에서는 규칙 기반 점수로 시작하고, 이후 AI 분석으로 확장한다.

## 7. 경기 기록

- 입력 권한: Owner, Manager
- 기록 항목: 승/무/패, 골, 도움, MVP
- 데이터 신뢰 정책: 운영자 입력을 공식 기록으로 보고 수정 이력을 저장
