# PROJECT_CONTEXT

## 1. 프로젝트 비전

MoIja는 축구/풋살 동호회 운영을 위한 SaaS 플랫폼이다. 단순히 경기를 만들고 인원을 모으는 앱이 아니라, 출석 신뢰도와 경기 데이터를 기반으로 팀 운영 수준을 높이는 도구다.

서비스의 출발점은 운영자가 경기일마다 회원과 용병의 참석 여부를 안정적으로 관리하는 것이다. 그 다음 단계는 출석 데이터, 선수 정보, 포메이션, 경기 결과를 연결해 팀의 승률과 경기력을 높이는 것이다. 장기적으로는 운영자와 코치가 경기 전후 또는 경기 중에 더 나은 판단을 하도록 돕는 AI Coach로 확장한다.

### 핵심 목표

- 참여율 향상
- 노쇼 감소
- 출석 신뢰도 기반 팀 문화 형성
- 운영자 반복 업무 최소화
- 회원이 자신의 참석률과 신뢰도를 인식하도록 유도
- 경기 기록과 포지션 데이터를 통한 팀 경기력 향상
- 데이터 기반 전술 분석과 운영 의사결정 지원

### 서비스 성장 단계

| 단계 | 목표 | 주요 기능 |
| --- | --- | --- |
| Phase 1. 운영 안정화 | 경기 참석 관리와 노쇼 감소 | 팀 생성, 멤버 초대, 경기 생성, 참석 신청, 운영자 출석 확정, GPS/QR 체크, 신뢰도 |
| Phase 2. 팀 데이터화 | 누가 잘 참석하고 어떻게 뛰는지 기록 | 참석률, 노쇼율, 연속 참석, 포지션, 주발, 골/도움/MVP, 시즌 통계 |
| Phase 3. 경기력 향상 | 포메이션과 선수 배치가 승률에 미치는 영향 분석 | 라인업, 포메이션 보드, 경기별 포지션 기록, 전략별 승률 |
| Phase 4. AI Coach | 감독/운영자의 전술 판단 보조 | 추천 포메이션, 대체 선수, 불리한 구역 분석, 경기 운영 제안 |

### 서비스 방향

- 초기 타겟: 축구, 풋살 동호회
- 핵심 사용자: Owner, Manager, Coach, Member, Guest
- 향후 확장: 러닝 크루, 스터디 모임, 독서 모임, 기타 정기 모임
- 설계 원칙: DB 및 시스템 구조는 정기 모임 운영에 재사용 가능하게 만들되, 축구/풋살의 포지션/전술 데이터는 확장 모듈로 분리한다.

## 3. 사용자 역할 및 권한

| 역할 | 권한 |
| --- | --- |
| Owner | 팀 생성/설정, 권한 관리, 경기 생성, 출석 확정, 경기 결과 입력, 공식 기록 확정 |
| Manager | 경기 생성, 참석/대기/용병 관리, 출석 확정, 경기 결과 입력 |
| Coach | 포지션 지정, 라인업 구성, 포메이션 저장, 선수 배치와 전술 기록 관리 |
| Member | 참석 신청, 출석 체크, 개인 참석률/신뢰도/기록 조회, 본인 포지션/주발 정보 입력 |
| Guest | 초대받은 특정 경기 참석, 제한된 출석 체크, 해당 경기 정보 확인 |

Guest는 팀 멤버가 아니다. 용병이나 외부 참석자는 경기 단위 초대 코드 또는 링크로 들어오며, 팀 내부 데이터 접근 권한은 최소화한다.

## 4. 로그인 및 개인정보 정책

- 기본 방식: Kakao 로그인
- 저장 정보: Kakao ID, 닉네임, 프로필 이미지
- 선수/참석 정보: 포지션, 가능 포지션, 주발, 참석률, 경기 기록
- Guest 최소 정보: 초대명 또는 표시명, 임시 프로필, 경기 참석 상태
- 미저장 정보: 전화번호, 주민등록번호, 주소, 생년월일
- 위치 정보: GPS 출석 검증에 필요한 순간의 좌표와 판정 결과만 저장하고, 상시 위치 추적은 하지 않는다.

## 5. 출석 시스템

### 지원 방식

| 방식 | 설명 |
| --- | --- |
| 수동 출석 | 운영자가 직접 참석, 불참, 노쇼를 확정 |
| QR 출석 | 경기장 도착 후 QR 스캔으로 체크인 |
| GPS 출석 | 경기 장소 반경 안에 있는지 확인 |
| GPS + 운영자 승인 | GPS 확인 후 운영자가 최종 승인 |

### 출석 상태

- 미응답
- 참석 예정
- 대기
- 불참
- 체크인 완료
- 운영자 확정 참석
- 노쇼
- 지각
- 취소

MVP에서는 `attending`, `absent`, `waitlisted`, `no_show`를 우선 사용하되, 이후 체크인과 운영자 확정 상태를 분리할 수 있어야 한다.

### 장소 관리

- 자유 텍스트 입력은 최소화한다.
- Kakao Maps 또는 Naver Maps 기반 장소 선택을 우선한다.
- 저장 데이터: 장소명, 주소, 위도, 경도, 출석 인정 반경
- 홈구장 즐겨찾기를 지원한다.

## 6. 초대와 용병 관리

팀 초대와 경기 초대는 구분한다.

- 팀 초대: 멤버로 가입시키는 코드 또는 링크
- 경기 초대: 회원이 아닌 용병을 특정 경기 참석자로 초대하는 코드 또는 링크
- 경기 초대는 만료 시간, 사용 횟수, 대상 경기, 초대자, 참석 상태를 가져야 한다.
- 용병 기록은 팀 시즌 통계와 분리하되, 경기 결과와 포지션 분석에는 포함할 수 있어야 한다.

## 7. 신뢰도 시스템

신뢰도는 참석률만이 아니라 약속을 얼마나 안정적으로 지키는지를 보여주는 지표다.

측정 요소:

- 참석률
- 노쇼율
- 연속 출석
- 지각
- 취소 시점
- 대기 전환 후 실제 참석
- 운영자 확정 기록

MVP에서는 규칙 기반 점수로 시작한다. AI 분석은 점수가 쌓인 뒤 위험 징후, 리마인드 추천, 멤버별 참석 패턴 분석으로 확장한다.

## 8. 선수 정보와 전술 데이터

회원 및 참석자는 다음 축구/풋살 데이터를 가질 수 있다.

- 주 포지션
- 가능 포지션 복수 선택
- 주로 쓰는 발: left, right, both
- 선호 플레이 성향
- 경기별 실제 배치 포지션
- 경기별 골, 도움, MVP, 기타 기록

운영자 또는 코치는 경기 전에 포메이션을 구성하고 작전판 형태로 저장한다. 저장된 라인업은 경기 참석자에게 공유되어 경기 전에 자신의 위치와 역할을 확인할 수 있어야 한다.

## 9. 경기 기록과 승률 분석

- 입력 권한: Owner, Manager
- 전술/라인업 권한: Owner, Manager, Coach
- 기록 항목: 승/무/패, 득점, 실점, 골, 도움, MVP, 포지션, 포메이션, 상대팀, 경기 메모
- 데이터 신뢰 정책: 운영자 입력을 공식 기록으로 보고 수정 이력을 저장한다.
- 분석 목표: 어떤 선수 조합, 포지션 배치, 포메이션 전략일 때 승률과 득점력이 높았는지 계산한다.

## 10. AI Coach 방향

AI Coach는 MVP 기능이 아니다. 다만 데이터 모델은 AI Coach가 나중에 답변할 수 있게 준비해야 한다.

예시:

- 이번 경기 참석자 기준 추천 포메이션은 무엇인가?
- 최근 승률이 높은 선수 조합은 누구인가?
- 특정 포지션이 약할 때 대체 가능한 멤버는 누구인가?
- 상대팀과 다시 경기할 때 이전에 효과적이었던 전략은 무엇인가?
- 경기 중 특정 측면이 불리할 때 어떤 포지션 이동이 도움이 되는가?

AI 추천은 반드시 근거 데이터를 함께 보여줘야 한다. 예: 최근 5경기, 해당 포지션 출전 기록, 승률, 득실 차, 참석 안정성.
