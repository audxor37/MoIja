# AI_ROADMAP

AI 기능은 MVP가 아니다. 먼저 운영 데이터와 경기 데이터가 구조화되어야 한다. MoIja의 AI는 사용자를 대신해 공식 기록을 바꾸는 기능이 아니라, 운영자와 Coach가 판단할 수 있도록 근거 있는 제안을 제공하는 기능이다.

## Phase 1: 규칙 기반 운영 분석

- 출석률 계산
- 노쇼율 계산
- 연속 출석 계산
- 마감 직전 취소 감지
- 대기자 전환 성공률 계산
- 신뢰도 점수 산정
- 노쇼 위험 멤버 표시

## Phase 2: 경기 데이터 분석

- 포지션별 경기 수와 승률
- 포지션별 득점/도움/MVP
- 멤버별 주 포지션과 실제 배치 포지션 비교
- 포메이션별 승률과 득실 차
- 선수 조합별 결과
- 상대팀별 이전 전적과 사용 전략

## Phase 3: AI Assistant

운영자가 자연어로 팀 상태를 물어보는 단계다.

예시 질문:

- "이번 달 노쇼 위험이 높은 멤버는?"
- "참석률이 떨어진 멤버에게 어떤 알림을 보내면 좋을까?"
- "최근 10경기에서 CAM 성과가 가장 좋은 멤버는?"
- "이번 경기 참석자 중 수비 포지션이 가능한 사람은?"
- "대기자 중 출석 신뢰도가 높은 사람은?"

## Phase 4: AI Coach

경기 전 라인업과 포메이션을 추천하는 단계다.

기능:

- 추천 포메이션
- 라인업 후보
- 포지션별 대체 멤버
- 주발과 포지션 균형 확인
- 참석 신뢰도를 반영한 선발 추천
- 상대팀 기록 기반 전략 추천

추천은 반드시 근거를 포함한다.

예시 근거:

- 최근 5경기 승률
- 해당 포지션 출전 기록
- 포메이션별 득실 차
- 멤버 참석 신뢰도
- 상대팀과의 이전 경기 결과

## Phase 5: In-game Coach

경기 중 운영자 또는 감독이 흐름을 보고 포메이션을 조정할 때 도움을 주는 단계다.

기능:

- 불리한 포지션 또는 측면 메모
- 유리한 구역 메모
- 실점 원인 태그
- 경기 중 포지션 이동 저장
- 포메이션 변경 후보 제안
- 대체 선수 또는 위치 변경 제안

입력 데이터:

- 현재 라인업
- 현재 포메이션
- 경기 시간
- 득점/실점 상황
- 운영자 메모
- 이전 경기의 유사 상황 기록

주의:

- 실시간 자동 판단은 초기 범위가 아니다.
- MVP 이후에도 운영자 입력 기반 분석부터 시작한다.
- AI 추천은 “판단 보조”이며 자동 전술 변경이 아니다.

## Phase 6: AI Club Manager

팀 운영 리포트와 유료 플랜용 인사이트 단계다.

- 참석률 트렌드 분석
- 운영자용 주간 요약
- 노쇼 예방 알림 추천
- 시즌별 성장 리포트
- 포지션 공백 리포트
- 용병 의존도 분석
- 유료 플랜용 인사이트 리포트

## AI 설계 원칙

- 원본 기록과 AI 분석 결과를 분리한다.
- AI 추천에는 근거 데이터를 함께 표시한다.
- AI가 공식 출석, 공식 기록, 공식 라인업을 직접 확정하지 않는다.
- 데이터가 부족하면 추천하지 않고 부족한 데이터를 명확히 말한다.
- 처음에는 규칙 기반 분석으로 충분히 검증한 뒤 AI를 붙인다.
