# DEVELOPMENT_RULES

## Core Rules

- 모바일 우선으로 구현한다.
- MVP에서는 AI보다 규칙 기반 로직을 우선한다.
- 개인정보는 최소 저장한다.
- Supabase RLS를 전제로 권한을 설계한다.
- Owner, Manager, Coach, Member, Guest 권한을 분리한다.
- 참석 신청과 실제 출석 확정은 분리한다.
- 팀 멤버와 경기 단위 용병은 분리한다.
- 출석, 권한, 기록 변경은 이력을 남긴다.
- AI 추천은 공식 기록을 덮어쓰지 않는다.
- GPS는 상시 추적이 아니라 출석 검증 이벤트로만 사용한다.

## Feature Checklist

새 기능은 아래 중 하나 이상에 연결되어야 한다.

- 운영자 반복 업무를 줄이는가?
- 참석률을 높이는가?
- 노쇼를 줄이는가?
- 멤버가 본인 참석 상태를 더 잘 이해하게 하는가?
- 신뢰도/랭킹/시즌 통계에 필요한 데이터를 남기는가?
- 경기 기록/포지션/포메이션 분석에 필요한 데이터를 남기는가?
- 개인정보 저장을 최소화하는가?

연결되지 않으면 구현하지 말고 보류한다.

## Role Rules

| 역할 | 가능 작업 |
| --- | --- |
| Owner | 팀 전체 관리, 권한 관리, 경기/출석/기록 관리 |
| Manager | 경기 생성, 참석 확정, 용병 관리, 결과 입력 |
| Coach | 라인업, 포지션, 포메이션 관리 |
| Member | 본인 참석 응답, 출석 체크, 개인 정보 수정 |
| Guest | 초대받은 경기 참석 응답 |

## Permission Rules

- Member는 공식 출석 확정, 노쇼 처리, 경기 결과 입력을 할 수 없다.
- Coach는 팀 전체 권한을 갖지 않는다.
- Guest는 팀 멤버 목록, 내부 랭킹, 시즌 통계에 접근할 수 없다.
- 출석 확정과 노쇼 처리는 Owner 또는 Manager만 가능하다.
- 공식 경기 기록은 Owner 또는 Manager가 입력한다.

## MVP Order

1. 인증
2. 팀 생성
3. 팀 멤버 초대
4. 경기 생성
5. 참석 신청
6. 운영자 출석 확정
7. 팀 초대 코드
8. 경기 단위 용병 초대
9. 신뢰도 계산
10. 랭킹
11. 경기 결과와 개인 기록

## UX Rules

- 모바일에서 핵심 행동은 1~2번 안에 가능해야 한다.
- 운영자는 경기 당일 해야 할 일을 먼저 봐야 한다.
- 멤버는 본인의 참석 상태, 참석률, 신뢰도를 바로 이해해야 한다.
- 용병은 팀 가입 없이 참석 응답이 가능해야 한다.
- 버튼과 상태명은 한국 사용자가 바로 이해할 수 있어야 한다.

## Data Rules

- 최종 상태만 저장하지 말고 변경 이력을 남긴다.
- 참석 의사와 실제 출석 확정은 분리한다.
- 멤버와 용병은 구분하되 경기 참석자로 함께 다룰 수 있어야 한다.
- 공식 기록, 자동 계산, AI 추천은 구분한다.
- 분석용 데이터는 원본 이벤트를 삭제하지 않는다.
