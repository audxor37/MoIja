# DEVELOPMENT_RULES

## 제품 판단 기준

1. 운영자의 반복 업무를 줄이는가?
2. 회원의 참석 행동을 더 명확하게 만드는가?
3. 노쇼와 늦은 취소를 줄이는가?
4. 축구/풋살에 유용하지만 다른 정기 모임에도 확장 가능한가?
5. 개인정보를 최소한으로 저장하는가?

## 개발 원칙

- 모바일 우선으로 설계한다.
- MVP에서는 규칙 기반 로직을 우선한다.
- Supabase Row Level Security를 전제로 권한을 설계한다.
- 출석, 경기 기록, 권한 변경처럼 신뢰가 필요한 데이터는 변경 이력을 남긴다.
- 스포츠 종속 필드는 가능한 한 `sport_type`, `position_code`처럼 확장 가능한 코드값으로 둔다.

## 역할별 권한 원칙

- Owner: 모든 팀 운영 권한
- Manager: 경기와 출석 운영 권한
- Coach: 포지션과 라인업 관리 권한
- Member: 본인 참석 신청과 개인 데이터 조회 권한

## MVP 우선순위

1. 인증과 팀 생성
2. 멤버 초대/권한
3. 경기 생성
4. 참석 신청
5. 출석 확정
6. 신뢰도 계산
7. 랭킹
8. 경기 결과와 개인 기록

## 비기능 요구사항

- 개인정보 최소 저장
- 변경 이력 보존
- 모바일 하단 내비게이션
- 느린 네트워크에서도 핵심 화면이 먼저 보이도록 구성
